برنامه‌نویسی شی‌گرا در پایتون

درک کلاس‌ها، اشیاء و اصول OOP برای ساخت برنامه‌های قدرتمندتر.

برنامه‌نویسی شی‌گرا (OOP) چیست؟

برنامه‌نویسی شی‌گرا (Object-Oriented Programming - OOP) یک پارادایم برنامه‌نویسی است که بر مفهوم "شیء" مبتنی است. این اشیاء می‌توانند حاوی داده‌ها (ویژگی‌ها/properties) و کد (متدها/methods) باشند. هدف اصلی OOP سازماندهی کد به گونه‌ای است که مدل‌سازی موجودیت‌های دنیای واقعی را آسان‌تر کند و قابلیت استفاده مجدد، نگهداری و مقیاس‌پذیری کد را افزایش دهد.

چهار اصل اساسی OOP عبارتند از:

  • کپسوله‌سازی (Encapsulation): بسته‌بندی داده‌ها و متدهایی که بر روی آن داده‌ها عمل می‌کنند در یک واحد (کلاس).
  • وراثت (Inheritance): توانایی یک کلاس (کلاس فرزند) برای به ارث بردن ویژگی‌ها و متدها از کلاس دیگر (کلاس والد).
  • چندریختی (Polymorphism): توانایی اشیاء مختلف برای به اشتراک گذاشتن یک رابط مشترک، اما پیاده‌سازی متفاوتی از آن.
  • انتزاع (Abstraction): نمایش تنها اطلاعات ضروری و پنهان کردن جزئیات پیاده‌سازی پیچیده.

کلاس‌ها و اشیاء

کلاس (Class)

یک کلاس مانند یک طرح (blueprint) یا قالب برای ایجاد اشیاء است. این یک ساختار منطقی است که ویژگی‌ها (متغیرها) و رفتارها (توابع/متدها) را تعریف می‌کند که اشیاء ایجاد شده از آن کلاس خواهند داشت.

برای تعریف یک کلاس از کلمه کلیدی class استفاده می‌کنیم:


class Car:
    pass # یک کلاس خالی برای شروع

شیء (Object)

یک شیء (که به آن نمونه نیز گفته می‌شود) یک نمونه از یک کلاس است. وقتی کلاسی را تعریف می‌کنید، فقط یک طرح ایجاد می‌کنید. برای استفاده از کلاس، باید یک شیء (نمونه) از آن ایجاد کنید.


my_car = Car() # ایجاد یک شیء (نمونه) از کلاس Car
print(my_car)

ویژگی‌ها و متدها

ویژگی‌ها (Attributes)

ویژگی‌ها متغیرهایی هستند که داده‌ها را برای یک شیء خاص نگه می‌دارند. هر شیء می‌تواند مقادیر ویژگی‌های خاص خود را داشته باشد.

متدها (Methods)

متدها توابعی هستند که در داخل یک کلاس تعریف می‌شوند و بر روی داده‌های مربوط به شیء عمل می‌کنند. آنها رفتارهای یک شیء را تعریف می‌کنند.

سازنده __init__

متد __init__ یک متد خاص است که به آن سازنده (constructor) گفته می‌شود. این متد به صورت خودکار هر زمان که یک شیء جدید از کلاس ایجاد می‌شود، فراخوانی می‌شود. از آن برای مقداردهی اولیه ویژگی‌های شیء استفاده می‌شود.

پارامتر self در متدهای کلاس به نمونه فعلی کلاس اشاره دارد و باید اولین پارامتر هر متد باشد.


class Dog:
    # سازنده کلاس
    def __init__(self, name, age):
        self.name = name # ویژگی
        self.age = age   # ویژگی

    # متد
    def bark(self):
        return f"{self.name} واق واق می کند!"

# ایجاد اشیاء از کلاس Dog
dog1 = Dog("لوسی", 3)
dog2 = Dog("ماکس", 5)

print(f"{dog1.name} {dog1.age} ساله است.")
print(f"{dog2.name} {dog2.age} ساله است.")
print(dog1.bark())

خروجی:


لوسی 3 ساله است.
ماکس 5 ساله است.
لوسی واق واق می کند!

وراثت (Inheritance)

وراثت به یک کلاس اجازه می‌دهد تا ویژگی‌ها و متدهای کلاس دیگری را به ارث ببرد. کلاسی که به ارث می‌برد (کلاس فرزند/مشتق) می‌تواند ویژگی‌ها و متدهای جدیدی اضافه کند یا متدهای والد را بازنویسی (override) کند. این کار قابلیت استفاده مجدد از کد را افزایش می‌دهد.


# کلاس والد
class Animal:
    def __init__(self, name):
        self.name = name

    def speak(self):
        pass # متد انتزاعی یا پایه

# کلاس فرزند از Animal ارث می‌برد
class Cat(Animal):
    def speak(self):
        return f"{self.name} میو میو می کند."

class Cow(Animal):
    def speak(self):
        return f"{self.name} ماغ می کشد."

cat = Cat("پیشی")
cow = Cow("گاوی")

print(cat.speak())
print(cow.speak())

خروجی:


پیشی میو میو می کند.
گاوی ماغ می کشد.

چندریختی (Polymorphism)

چندریختی به معنی "اشکال مختلف" است. در OOP، این اصل به متدهای مختلفی (در کلاس‌های مختلف) با نام یکسان اشاره دارد که می‌توانند رفتارهای متفاوتی را بسته به شیئی که آن‌ها را فراخوانی می‌کند، از خود نشان دهند. مثال بالا از speak() یک نمونه ساده از چندریختی است.

کپسوله‌سازی (Encapsulation)

کپسوله‌سازی به مفهوم بسته‌بندی داده‌ها و متدهایی که بر روی آن داده‌ها عمل می‌کنند در یک واحد (کلاس) اشاره دارد. همچنین، به پنهان کردن جزئیات داخلی یک شیء و کنترل دسترسی به داده‌ها از خارج از کلاس کمک می‌کند.

در پایتون، هیچ مفهوم private یا public سختگیرانه‌ای وجود ندارد. اما با استفاده از پیشوندهای زیرخط (_ و __) می‌توانیم قصد خود را برای ویژگی‌های "محافظت شده" یا "خصوصی" نشان دهیم.


class BankAccount:
    def __init__(self, balance):
        self.__balance = balance # ویژگی "خصوصی" (با توافق)

    def deposit(self, amount):
        if amount > 0:
            self.__balance += amount
            print(f"واریز {amount} انجام شد. موجودی جدید: {self.__balance}")
        else:
            print("مبلغ واریزی باید مثبت باشد.")

    def get_balance(self):
        return self.__balance

account = BankAccount(100)
account.deposit(50)
print(f"موجودی فعلی: {account.get_balance()}")
# تلاش برای دسترسی مستقیم به ویژگی "خصوصی" (توصیه نمی شود)
# print(account.__balance) # این خطا خواهد داد
print(account._BankAccount__balance) # می توان به این روش دسترسی پیدا کرد اما توصیه نمی شود

انتزاع (Abstraction)

انتزاع به فرآیند پنهان کردن جزئیات پیاده‌سازی و نشان دادن تنها اطلاعات ضروری به کاربر (برنامه‌نویس) اشاره دارد. این کار پیچیدگی سیستم را کاهش می‌دهد و استفاده از آن را آسان‌تر می‌کند. در پایتون، می‌توان از کلاس‌های انتزاعی و متدهای انتزاعی (با استفاده از ماژول abc) برای اعمال انتزاع استفاده کرد.


from abc import ABC, abstractmethod

class Shape(ABC): # کلاس انتزاعی
    @abstractmethod
    def area(self):
        pass

    @abstractmethod
    def perimeter(self):
        pass

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius

    def area(self):
        return 3.14 * self.radius * self.radius

    def perimeter(self):
        return 2 * 3.14 * self.radius

# shape = Shape() # نمی توانید از یک کلاس انتزاعی نمونه بسازید

circle = Circle(5)
print(f"مساحت دایره: {circle.area()}")
print(f"محیط دایره: {circle.perimeter()}")

خروجی:


مساحت دایره: 78.5
محیط دایره: 31.400000000000002
